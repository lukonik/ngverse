The world has changed since the AI revolution—and it continues to change rapidly. In the past, we relied on third-party libraries to integrate prebuilt components into our apps. Today, we often ask AI to generate these components on demand. While this can be powerful, it comes with a cost: AI-generated code is often unpredictable, inconsistent, and prone to subtle bugs.

That’s exactly what ngverse/ui aims to solve.

is a powerful Angular library packed with high-quality, production-ready components, directives, pipes animations—just to name a few. It’s built on top of @angular/cdk and Tailwind CSS to ensure everything is robust, flexible, and easy to customize.

Using feels just like copying code from an AI tool into your app—but with a big difference: everything here is tested, stable, and feature-rich.

How it works

includes Angular schematics that automatically add the source code of components, directives, and other UI features directly into your app.

For example, if you want to add a select component, just run:

ng g @ngverse/ui:ui select

This will generate the complete, fully functional source code for the select component—right inside your project.

You get complete control over the code: it’s customizable, tested, and ready to use—as if you had written it yourself, but without the hassle.

Dependencies

The core dependencies behind are:

  • @angular/cdk
  • Tailwind CSS

These two libraries form the backbone of most features in

However, in some edge cases, we rely on a few lightweight, low-level libraries—only when it makes the code more readable, stable, and avoids overly complex or “crypto-like” logic in your app.

For example:

  • The kebabCase pipe uses the popular change-case library to keep the implementation simple and reliable.
You only need to install The Optional dependencies when a specific feature requires them.
